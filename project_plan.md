# ResNet50 目标定向对抗攻击与交互式 Web 演示系统开发计划

## 项目总览

本项目旨在基于PyTorch框架，利用本地计算资源（NVIDIA RTX 5060 GPU），对官方预训练的ResNet50分类图像模型进行安全性验证 。 核心目标不仅在于复现快速梯度符号法（FGSM）以实现目标定向的对抗样本攻击 ，最终还将**交付一个交互式的 Web 应用程序**。用户可通过该网站上传自定义图像，通过可视化滑块动态调节扰动步长（$\epsilon$），并实时观察对抗噪声叠加后，模型从高置信度的原始分类错误识别为指定目标分类的动态过程 。

## 第一阶段：基线工程与底层模块化（Baseline & Modularization）

**目标**：确保 GPU 加速正常运行，完成标准的前向传播推理，并将核心逻辑封装为便于 Web 端调用的模块 。

- **任务 1.1：运行环境与算力验证**
  
  - **执行动作**：检查 Python 3.13 运行环境，确认已安装支持 CUDA 13.0 和 sm_120 架构的 PyTorch (Nightly) 版本，以消除 GPU 兼容性警告并释放 RTX 5060 的完整算力 。

- **任务 1.2：构建模型加载与推理基准**
  
  - **核心逻辑**：加载 `ResNet50_Weights.DEFAULT` 预训练权重 。动态获取 ImageNet-1K 标签映射表 。
  
  - **代码规范**：将模型加载逻辑封装，利用全局缓存机制（如 `@st.cache_resource`），确保在 Web 端多次推理时不会重复加载模型。
  
  - **基准测试**：遍历本地测试目录，输出原始预测的 Top-1/Top-5 类别及置信度（例：[dingo] (ID: 273) - 置信度: 38.95%） ，以此作为后续攻击的基准数据。

## 第二阶段：对抗攻击引擎开发（Attack Engine Implementation）

**目标**：编写并封装实现 FGSM 目标定向攻击的核心数学逻辑，确保其能接受 Web 端的动态参数传入 。

- **任务 2.1：攻击核心函数封装**
  
  - **执行动作**：引入 PyTorch 自动求导（Autograd）模块 。确保输入图像张量设置 `requires_grad = True` 。

- **任务 2.2：构建定向损失函数与梯度提取**
  
  - **核心逻辑**：将模型的输出与指定的目标类别 ID（Target ID，如 7 代表母鸡 ）进行交叉熵损失（Cross-Entropy Loss）计算 。
  
  - **梯度反传**：调用 `loss.backward()` 获取输入图像数据域的梯度（Data Gradient） 。

- **任务 2.3：生成对抗扰动（Perturbation）**
  
  - **核心逻辑**：使用公式 $X_{adv} = X - \epsilon \cdot \text{sign}(\nabla_X J(X, Y_{target}))$ 生成对抗样本 。
  
  - **参数接口**：将扰动步长参数 $\epsilon$ (Epsilon) 暴露为函数输入参数 ，用于接收 Web 端用户的滑动条输入，控制噪声强度并保持在人类视觉感知的阈值之下。

## 第三阶段：交互式 Web 网站开发（Interactive Web UI） **[新增阶段]**

**目标**：基于 Streamlit/Gradio 框架搭建前端网站，实现用户交互与对抗攻击过程的可视化。

- **任务 3.1：Web 页面布局设计**
  
  - **侧边栏控制区**：提供图片上传组件（支持本地图像上传）、目标类别下拉选择框（例如选择 Target ID 7 Hen ）、以及 $\epsilon$ 值动态滑块（范围例如从 0.00 到 0.10，步长 0.01）。
  
  - **主展示区**：采用多列布局，并排展示“原始图像”、“提取的梯度噪声图”和“合成后的对抗样本图像”。

- **任务 3.2：状态同步与实时渲染**
  
  - **核心逻辑**：捕捉滑块的变化事件，实时调用第二阶段的“对抗攻击引擎”，动态生成对抗张量并进行逆标准化（De-normalization）转为 RGB 像素值展示 。

- **任务 3.3：置信度变化可视化**
  
  - **执行动作**：将原始样本和对抗样本分别送入 `eval()` 模式的 ResNet50 进行推理 。
  
  - **预期输出**：使用柱状图（Bar Chart）直观对比原始分类（如 Dingo）置信度的显著下降，以及目标分类（如 Hen）置信度的跃升 。

## 第四阶段：代码重构与文档完善（Refactoring & Documentation）

**目标**：提升工程代码质量，完善学术化交付物 。

- **任务 4.1：系统性能调优与重构**
  
  - **执行动作**：将基线推理与 FGSM 攻击逻辑彻底解耦，统一封装为标准 Python 类（Class） 。优化 GPU 显存释放机制，防止 Web 端连续拖动滑块时导致 RTX 5060 显存溢出（OOM）。

- **任务 4.2：完善项目说明（README.md）**
  
  - **执行动作**：补充 Web 系统的启动命令（如 `streamlit run app.py`）、环境配置指南（特针针对 RTX 5060 的环境建议） 、核心算法简要说明 ，以及项目依赖文件（`requirements.txt`），完成仓库的标准搭建 。
